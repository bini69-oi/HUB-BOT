# Телеметрия ошибок

Каждая установка HUB-BOT шлёт необработанные ошибки на наш сервер приёма — мы видим
падения у владельцев раньше, чем они напишут в поддержку, и чиним по трейсу.

## Что видит юзер и владелец

Ошибка больше не роняет процесс и не оставляет юзера в тишине:

- **бот** — юзеру уходит «⚠️ Что-то пошло не так… Код ошибки: E1a2b3c-4d5e», бот жив;
- **web/API** — чистый JSON `{"ok": false, "detail": "внутренняя ошибка сервера",
  "error_id": "E…"}`, без стектрейса наружу;
- **воркер** — задача ретраится как раньше, ошибка дополнительно репортится.

Код ошибки юзер называет поддержке — по нему находим точный трейс на дашборде.

## Что отправляется

Класс исключения, сообщение (до 500 символов), трейс с обрезанными путями
(`File "src/…"` вместо абсолютных), источник (web/bot/worker), маленький контекст
(путь запроса / команда / имя задачи), версия приложения и анонимный id установки —
хэш от токена бота, необратимый. Данные юзеров, токены, конфиги — не уходят никогда.

Отправка fire-and-forget: фоновая задача, батчи раз в ~30 секунд, повторы одной и той
же ошибки схлопываются в счётчик, недоступность сервера приёма ни на что не влияет.

Выключается одной строкой в `.env`: `TELEMETRY__ENABLED=false`.

## Наш сервер приёма

Код в [telemetry-server/](../telemetry-server/) — самодостаточный, деплой в три
команды (`docker compose up -d --build` + nginx на `errors.<домен>`), инструкция в его
README. Дашборд: список проблем по fingerprint (счётчик, установки, версии, трейс,
resolve/reopen), Telegram-алерт на новую проблему и на регрессию решённой.

По умолчанию `TELEMETRY__URL` пуст — телеметрия не шлёт ничего, пока не пропишешь
адрес своего сервера приёма (`TELEMETRY__URL=https://errors.<домен>/ingest`). Поднять
приём надо один раз на своей стороне.

## Куда смотреть в коде

- клиент: `src/infrastructure/services/telemetry.py` (репортёр),
  `src/web/app.py` (web-хендлер), `src/bot/errors.py` (бот),
  `src/infrastructure/taskiq/broker.py` (воркер);
- явный репорт из своего кода: `container.telemetry.report(exc, source="web",
  context={...})` — вернёт `error_id` для показа юзеру.
