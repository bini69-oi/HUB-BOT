# 📗 Инструкция оператору: с нуля до первых продаж

Эта страница — для владельца магазина, а не для программиста. Здесь всё, что нужно,
чтобы установить продукт на сервер, настроить его мышкой в веб-кабинете и начать
принимать деньги. Читать код не придётся: почти всё делается тумблерами и полями,
изменения применяются на лету, без перезапусков.

---

## 1. Что вы получаете

**VPN-HUB BOT** — готовый VPN-магазин поверх вашей панели **Remnawave**. В одном стеке:

- 🤖 **Telegram-бот** — продажи, пробный период, баланс-кошелёк, рефералка, тикеты. Меню бота собирается в конструкторе.
- 📱 **Мини-аппа** — личный кабинет подписчика внутри Telegram: статус, тарифы, оплата, подключение в приложение. **8 тем оформления** на выбор.
- 🌐 **Сайт-лендинг + веб-кабинет** — витрина и покупка прямо с сайта, по email, без Telegram (можно включить позже).
- 🖥 **Админ-кабинет** — веб-панель на `/admin/`: тарифы, платёжки, меню, мини-аппа, серверы, рассылки и 70+ настроек с мгновенным применением.
- 🧠 **ИИ-поддержка** — Claude отвечает клиентам в тикетах вместо вас, а сложные случаи (деньги, споры) передаёт человеку.

Всё это ставится **одной командой** и настраивается из кабинета. Панель Remnawave вы держите свою.

## 2. Что нужно перед установкой

| Что | Подробности |
|---|---|
| **VPS (сервер)** | Linux (Ubuntu/Debian). Рекомендуется **1 vCPU / 2 GB RAM**. На **1 GB** установщик сам поднимет 2 GB swap, чтобы сборка не упала по нехватке памяти. |
| **Домен** *(желательно)* | Нужен для HTTPS. Без домена стек поднимется на `http://IP`, но **кнопка мини-аппы в Telegram не появится** — Telegram открывает мини-аппы только по HTTPS. Для домена также спросят e-mail для бесплатного сертификата Let's Encrypt. |
| **Токен бота** | Создайте бота у [@BotFather](https://t.me/BotFather) → `/newbot` → скопируйте токен вида `1234:AA...`. |
| **Панель Remnawave** | Рабочая панель версии **≥ 2.8.0**: её URL + **API-токен**. Нет панели? Нажмите Enter при установке — поднимется встроенная мок-панель, чтобы всё пощупать. |

> Токен, ключи шифрования и пароли установщик генерирует и хранит сам — руками ничего в файлы вписывать не нужно.

## 3. Установка за 5 минут

Подключитесь к серверу по SSH и выполните две команды:

```bash
git clone https://github.com/bini69-oi/HUB-BOT.git && cd HUB-BOT
./scripts/install.sh
```

**Что спросит установщик** (только то, что нельзя придумать за вас):

1. **Токен бота** — из @BotFather (обязательно).
2. **Домен** — Enter, если домена нет (будет `http://IP`).
3. **E-mail для Let's Encrypt** — если указали домен.
4. **URL панели Remnawave** — Enter для встроенной мок-панели.
5. **API-токен панели** — если указали URL панели.

Дальше он всё делает сам: ставит Docker, при нехватке памяти поднимает swap, генерирует секреты,
собирает и запускает весь стек (postgres · redis · web · bot · worker · scheduler · caddy),
применяет миграции и ждёт, пока сервис оживёт.

**Что выдаст в конце:**

```
🎉 Готово!
   Кабинет      https://ваш-домен/admin/
   Логин        admin
   Пароль       <сгенерированный пароль>
   Мини-аппа    https://ваш-домен/app/
```

Запишите пароль — он лежит в файле `.env` (строка `ADMIN__PASSWORD`), но лучше сохранить сразу.

> Первая сборка занимает несколько минут — это нормально. Если что-то пошло не так, установщик покажет последние строки лога и путь к полному логу.

## 4. Первый вход в кабинет

- Откройте **`https://ваш-домен/admin/`** (или `http://IP/admin/`).
- Логин **`admin`**, пароль — из вывода установщика.
- **Демо-режим** (read-only, без пароля): если включён `ADMIN__DEMO_ENABLED=true` в `.env`, на странице входа будет кнопка одноразового демо-входа «только смотреть». Удобно показать кабинет партнёру, ничего не сломав. Для боевого магазина держите его **выключенным**.

**Смена пароля администратора.** Отдельной кнопки в интерфейсе нет — пароль задаётся в файле `.env`:

```bash
cd HUB-BOT
nano .env                 # поменяйте строку ADMIN__PASSWORD=...
./scripts/update.sh       # перезапустит стек и применит новый пароль
```

Новый пароль применится при следующем старте (система перехэширует его автоматически).

## 5. Go-live чеклист

Идите по порядку. Пометка **[минимум]** — без этого продавать нельзя; **[желательно]** — можно включить позже.

1. **[минимум] Подключить панель** — раздел **«Серверы»**. Нажмите **«Синхронизировать» (Синк)** — подтянутся ваши ноды из Remnawave. Отметьте нужные ноды флажком **«в продаже»**. Если синк падает с ошибкой — проверьте URL и токен панели в `.env`.
2. **[минимум] Создать тариф** — раздел **«Тарифы»**. Задайте название, сроки и цены (например 30 дней / 149 ₽), лимит устройств и трафика. **Важно:** привяжите к тарифу хотя бы один сквод панели (поле *«сквады»* / internal squads) — иначе при покупке клиент получит **«ошибку VPN-сервера»** (см. §9).
3. **[минимум] Включить кассу** — раздел **«Платежи»**. Проще всего стартовать с **Telegram Stars** (работает сразу, без ключей и вебхуков). Для приёма карт — включите **YooKassa** и впишите `shop_id` + `secret_key`, затем жмите **«Тест»** — кнопка делает реальную пробу API. Подробности — в §6.
4. **[желательно] Собрать меню бота** — раздел **«Меню»**. Дерево кнопок с живым превью. Есть готовое меню по умолчанию — можно оставить как есть или подправить. Кнопка **«Сбросить к стандартному»** вернёт заводское меню.
5. **[желательно] Настроить мини-аппу** — раздел **«Мини-аппа»**. Выберите одну из **8 тем** (Минимал, Прайват, Бадди, Нативный, Терминал, Журнал, Неон, Поп), акцентный цвет и заголовок. Нажмите **«Опубликовать»**.
6. **[минимум] Проверить, что бот жив** — бот запускается сам сразу после установки. Отправьте ему `/start`: должно прийти приветствие и кнопка «📱 Приложение». Убедитесь, что в настройках **не включён** «Режим техработ» (`MAINTENANCE_MODE`). В разделе **«Настройки»** пропишите свой Telegram ID в `ADMIN_IDS` и `BOT_USERNAME` (без @).
7. **[желательно] Пробная покупка** — купите тариф сами (Stars или ручное начисление баланса), убедитесь, что подписка выдалась и кнопка «Подключить» отдаёт ссылку. После теста — готово к продажам.

**Опционально, тумблерами** (применяется без перезапуска): пробный период (`TRIAL_ENABLED`, дни/трафик/устройства), рефералка (`REFERRAL_ENABLED`), промокоды и акции, автопродление с баланса, баннеры и логотип, покупка с сайта (`WEB_CABINET_ENABLED`).

## 6. Приём платежей

Кассы включаются в разделе **«Платежи»**: тумблер «активна» + поля ключей (шифруются) + кнопка **«Тест»**.
Как только касса активна — кнопки оплаты **сами появляются** в боте и мини-аппе. Клиент платит →
подписка выдаётся **автоматически по вебхуку**. Если вебхук потеряется, фоновый реконсилятор
каждые ~5 минут сам добьёт оплаченные счета.

| Касса | Что вписать | Вебхук нужен? |
|---|---|---|
| ⭐ **Telegram Stars** | ничего — работает из коробки | нет (подтверждение внутри Telegram) |
| 💼 **Вручную / баланс** | ничего (начисление админом) | — |
| 🏦 **YooKassa** (карта, СБП) | `shop_id`, `secret_key` (+ `recurrent_enabled` для автосписаний) | да |
| 🪙 **CryptoBot** (USDT/TON/BTC) | `api_token` из @CryptoBot | да |
| 🪙 **Cryptomus / Heleket** | `merchant_uuid`, `api_key` | да |
| 🏦 **Platega** (СБП/карта/крипта) | `merchant_id`, `secret`, `payment_method` | да |
| 👛 **ЮMoney (кошелёк)** | `wallet`, `notification_secret` | да |

Кроме них из коробки готовы: Robokassa, WATA, FreeKassa, PayPalych, CloudPayments, Lava,
MulenPay, KassaAI, RollyPay, RioPay, SeverPay, AuraPay, Antilopay — **21 живой провайдер**.

**Где взять вебхук-URL.** В личном кабинете провайдера укажите адрес вида:

```
https://ваш-домен/api/v1/payments/<касса>
```

Подставьте код кассы: `yookassa`, `cryptobot`, `cryptomus`, `platega`, `yoomoney` и т.д.
Пример для YooKassa: `https://ваш-домен/api/v1/payments/yookassa`.
Для **Telegram Stars** вебхук не нужен вовсе.

> Комиссия кассы (поле «комиссия») учитывается в расчёте чистой прибыли (оборот − комиссии − налог) в том же разделе.

## 7. ИИ-поддержка

Claude отвечает клиентам в тикетах, а разговоры про деньги и споры сам передаёт живому оператору.
Настраивается в разделе **«ИИ-поддержка»**:

1. **Вставьте свой ключ Claude** (`sk-ant-…`) — берётся на [console.anthropic.com](https://console.anthropic.com). Хранится в зашифрованном виде.
2. **Модель** — по умолчанию `claude-haiku-4-5` (дёшево и быстро для поддержки), менять не обязательно.
3. **Обучите базой знаний** — в поле «База знаний» опишите свой сервис своими словами: тарифы, какие приложения ставить, типовые решения. Пусто → возьмётся встроенный набор. Есть ещё поле «Доп. инструкции» для тона и стиля.
4. **Нажмите «Тест»** — задайте пробный вопрос и посмотрите, как ИИ отвечает, **до** включения на реальных клиентах.
5. Когда ответы устраивают — включите тумблер **«Включить ИИ-поддержку»**.

## 8. Обслуживание

- **Обновление одной кнопкой (авто).** Если вы указали свой Telegram ID при установке, бот сам раз в 6 ч проверяет GitHub и, когда выходит новая версия, присылает вам сообщение с кнопкой **«🔄 Обновить»**. Нажали — бот снимает бэкап БД, скачивает новую версию, пересобирается и перезапускается (пара минут). То же вручную: `/admin` → **🔄 Обновление**.
  - Автообновление выполняет сервис `updater` (включён по умолчанию, нужен доступ к docker-сокету — как у Watchtower). Выключить: уберите `updater` из `COMPOSE_PROFILES` в `.env`; тогда кнопка просто подскажет команду ниже.
- **Обновление вручную** — одной командой, безопасно:
  ```bash
  cd HUB-BOT && ./scripts/update.sh
  ```
  Скрипт **сначала снимает бэкап БД** в `backups/`, потом тянет обновления, пересобирает и мигрирует. Если новая версия не поднялась — печатает готовые команды отката. Данные не теряются.
- **Бэкапы** — раздел **«Обслуживание»** (и настройки категории «Бэкапы»): включить `BACKUP_ENABLED`, задать время `BACKUP_TIME` (по умолчанию 04:00), сколько копий хранить (`BACKUP_KEEP_LAST`) и **пароль шифрования** (`BACKUP_ENCRYPTION_PASSWORD`). Кнопка **«Сделать бэкап сейчас»** снимет копию вручную. Плюс `update.sh` всегда кладёт дамп в `backups/` перед обновлением.
  - ⚠️ **Пароль бэкапа** установщик печатает в конце установки — **сохраните его отдельно от сервера**. Без него зашифрованный архив не открыть. Если не задавали свой — это первые 16 символов `APP__JWT_SECRET` из `.env`. Копия архива также приходит вам в Telegram (если указан ваш ID), чтобы бэкап пережил потерю сервера.
  - **Восстановление из бэкапа** (на чистом сервере):
    ```bash
    # 1) распаковать архив (спросит пароль шифрования)
    unzip backup_YYYYMMDD_HHMMSS.zip        # → db_YYYYMMDD_HHMMSS.sql
    # 2) поднять только БД и залить дамп
    docker compose -f docker/compose.prod.yml up -d postgres
    cat db_*.sql | docker compose -f docker/compose.prod.yml exec -T postgres psql -U vpn -d vpn
    # 3) поднять остальной стек
    docker compose -f docker/compose.prod.yml up -d
    ```
- **Тех-режим** — тумблер «Режим техработ» (`MAINTENANCE_MODE`): бот отвечает всем заглушкой, кроме админов. Есть и **авто-техрежим** (`AUTO_MAINTENANCE_ENABLED`): если панель падает — бот сам включает заглушку, а когда панель вернётся — снимает.
- **Где смотреть здоровье** — `https://ваш-домен/health` отдаёт `200` + статус БД/Redis (лёгкий пинг, можно дёргать часто). Для мониторинга-аптайма есть `https://ваш-домен/health/deep` — добавляет статус **воркера** (фоновые задачи) и **панели**; возвращает `503`, если воркер мёртв (это важно — обычный `/health` этого не видит). Наведите на него ваш uptime-монитор (UptimeRobot и т.п.). На **дашборде** кабинета — выручка, подписки, онлайн и лента аудита.

## 9. Частые проблемы

| Симптом | В чём дело / что делать |
|---|---|
| **Кабинет не открылся сразу** | Сервису нужно ~15–20 секунд на старт (uvicorn + миграции). Подождите и обновите страницу. Если долго — `docker compose -f docker/compose.prod.yml logs web`. |
| **«Ошибка VPN-сервера» при покупке** | К тарифу **не привязан сквод** панели. Откройте «Тарифы» → тариф → поле «сквады» → выберите сквод Remnawave. Проверьте, что в «Серверы» ноды синхронизированы и отмечены «в продаже». |
| **Касса не появилась в боте** | Она **не активна** или **не заполнены поля**. Раздел «Платежи»: включите тумблер и впишите все ключи. Нажмите «Тест» — он подскажет, чего не хватает. |
| **Мало памяти / сборка падает** | На 1 GB RAM установщик сам поднимает 2 GB swap. Если ставили руками или swap не создался — добавьте его и повторите `./scripts/install.sh` (скрипт идемпотентен). |
| **Кнопка мини-аппы не появилась** | Мини-аппа требует HTTPS. Убедитесь, что ставили **с доменом** (не `http://IP`) и в настройках заполнен `SUBSCRIPTION_MINI_APP_URL`. |

## 10. Безопасность перед продажей клиентам

- **Смените дефолтный пароль администратора** — не оставляйте сгенерированный. Меняется через `.env` → `ADMIN__PASSWORD` + `./scripts/update.sh` (см. §4). Выключите демо-режим (`ADMIN__DEMO_ENABLED=false`).
- **Отдельные ключи уже сгенерированы установщиком** — `APP__CRYPT_KEY` (шифрование секретов), `APP__JWT_SECRET` (сессии кабинета), пароль БД, вебхук-секрет. Они уникальны для вашей установки — не копируйте чужие и не переиспользуйте между серверами.
- **Не коммитьте и не пересылайте `.env`** — в нём все секреты. Файл создаётся с правами `600` (только владелец). Держите его на сервере, не выкладывайте в git и чаты.
- **Ротация ключа Claude** — если ключ ИИ-поддержки утёк или вы отдаёте проект другому, отзовите старый на console.anthropic.com и впишите новый в разделе «ИИ-поддержка».
- **Ключи касс шифруются** в базе автоматически (Fernet) — в кабинете и API они никогда не показываются обратно, только отмечается, что поле заполнено.
