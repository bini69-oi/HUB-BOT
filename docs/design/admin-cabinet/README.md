# Handoff: ADVANCED ADMIN CABINET — веб-кабинет администратора VPN-бота

## Overview
Веб-панель администратора для Telegram-бота по продаже VPN-подписок (бэкенд на
**Remnawave**). Цель — «конструктор всего»: любой человек через UI чинит / меняет /
добавляет / убирает в боте всё — тарифы, акции, рассылки, пользователей, платёжные
системы, кнопки бота, Miniapp, серверы, 200+ настроек — без правки кода и без
перезапуска (hot-reload).

## About the Design Files
Файл в этом бандле — **дизайн-референс, сделанный в HTML** (интерактивный прототип на
React-подобной рантайм-обёртке). Это НЕ продакшен-код для копирования. Задача — 
**воссоздать эти экраны в реальном кодовом окружении проекта**, используя его стек и 
паттерны. Если фронта ещё нет — рекомендуемый стек ниже. Вся «логика» в прототипе на
мок-данных; в реальном проекте её заменяют вызовы REST API к бэкенду бота.

Файл: `VPN Admin Cabinet.dc.html` — открыть в браузере, чтобы видеть поведение 1:1
(темы, переключение языка, модалки, живые прогресс-бары, фильтры, поиск).

## Fidelity
**High-fidelity.** Финальные цвета, типографика, отступы, состояния. Воссоздавать
пиксельно. Точные значения — в разделе Design Tokens.

## Рекомендуемый стек (если фронта нет)
- **Frontend:** React 18 + TypeScript + Vite. Роутинг — react-router. Состояние
  сервера — TanStack Query (react-query). Локальный UI-стейт — useState/zustand.
  Стили — CSS-переменные для тем (как в прототипе) + CSS Modules или Tailwind
  (с кастомными токенами). Шрифты — Google Fonts: **Golos Text** (400/500/600) и
  **JetBrains Mono** (400/500/700).
- **Backend (уже существует у бота):** Python (aiogram + FastAPI), PostgreSQL, Redis.
  Веб-админка бота отдаёт REST API,
  аутентификация по API-токену. Кабинет — SPA поверх этого API. Многие настройки
  живут в таблице `bot_config` и применяются через hot-reload сервис.
- **Интеграция:** Remnawave API (узлы, сквады, подписки, трафик, HWID), синхронизация
  каждые ~15 мин. 18 платёжных провайдеров через единый вебхук-сервер (порт 8080).

## DESIGN TOKENS

### Темы (CSS-переменные на `body[data-theme]`, переключатель DARK/LIGHT, persist в localStorage)
DARK (по умолчанию):
```
--bg:#0A0A0A  --panel:#111111  --panel2:#171717
--border:#212121  --border2:#2E2E2E
--text:#F5F5F5  --muted:#8F8F8F  --dim:#5C5C5C
--pill:#1C1C1C  --hover:#161616  --inv:#0A0A0A
```
LIGHT:
```
--bg:#F5F5F5  --panel:#FFFFFF  --panel2:#FAFAFA
--border:#E4E4E4  --border2:#D2D2D2
--text:#141414  --muted:#6E6E6E  --dim:#A3A3A3
--pill:#ECECEC  --hover:#F0F0F0  --inv:#FFFFFF
```
Переход темы: `transition:background .2s`. Правило: интерфейс СТРОГО чёрно-белый,
никаких цветных акцентов — статусы передаются оттенками серого (online/успех = `text`,
ошибка/выкл = `dim`). Единственное исключение цвета — пользовательский выбор цвета
кнопки бота и акцента Miniapp (там HEX от админа).

### Типографика
- Заголовки/тело: **Golos Text**. H1 раздела 22px/600, letter-spacing −.01em.
  Тело 13–14px. Подписи 11–12px `--muted`.
- Всё «техническое»: **JetBrains Mono** — цифры, суммы, ID, %, статусы, ключи настроек,
  капс-метки. Капс-метка: 8.5–10px, letter-spacing .1em, `--dim`, обычно UPPERCASE.

### Радиусы / границы / тени
- Карточка: `--panel` + `1px solid --border` + `border-radius:4px` + padding 18–22px.
- Кнопки/инпуты/сегменты: radius 3px. Пилюли-капсулы: 99px.
- Теней почти нет; у модалок и drawer — `0 8px 24px rgba(0,0,0,.35)`.
- Затемнение оверлеев: `rgba(0,0,0,.55–.6)`.

### Примитивы (переиспользуются везде)
- **Toggle:** трек 34×19px, radius 10px, кружок 15px (`--bg`), `top/left:2px`,
  вкл → `transform:translateX(15px)`, фон вкл `--text` / выкл `--border2`,
  `transition:transform .15s`. (Крупный вариант в детале настроек — 46×26, кружок 20.)
- **Segmented control:** ряд ячеек в общей рамке `--border2` radius 3px; активная —
  фон `--text`, текст `--inv`; прочие — прозрачные, `--muted`; разделитель
  `border-left:1px solid --border`. Все ячейки/кнопки/табы — `white-space:nowrap`.
- **KPI-плитка:** капс-метка (mono, dim) → крупное моно-число 26px → подпись.
- **Мини-бар-чарт:** флекс столбиков разной высоты %, последний `--text`, прочие
  `--border2`, hover → `--muted`.
- **Toast:** `position:fixed; bottom-right; z-index:300`, фон `--text`, текст `--inv`,
  анимация `fadeUp .2s`, автоскрытие ~2.6с. На КАЖДОЕ действие.
- **Кнопка primary:** фон `--text`, текст `--inv`, hover `opacity:.85`.
- **Кнопка secondary:** прозрачная, `1px solid --border2`, hover `background --hover`.

### Анимации (keyframes)
`spin` (360°), `fadeUp` (opacity+translateY 8px), `fadeIn`, `slideIn` (translateX 32px,
для drawer), `pulse` (мигание точки статуса 2.4s).

### Адаптив
Все двухколоночные блоки складываются в одну колонку без горизонтального скролла:
`display:flex; flex-wrap:wrap`, главная колонка `flex:9999 1 420px`, боковая
`flex:1 1 300px`, у всех детей `min-width:0`. Сетки карточек —
`grid-template-columns:repeat(auto-fit,minmax(230px,1fr))`.

## INTERACTIONS & BEHAVIOR
- **Навигация:** клиентский роутинг по 14 разделам; активный пункт `background:--pill`.
- **Темы/язык:** DARK/LIGHT и RU/EN переключаются вживую, persist в localStorage
  (`data-theme` на body; словари RU/EN для всех строк). Реализовать i18n.
- **Toast** на каждое действие (создание, вкл/выкл, сохранение, копирование, ошибки).
- **Модалки:** промокод, кампания, план — overlay `rgba(0,0,0,.6)`, `fadeUp`, клик по
  фону и **ESC** закрывают, клик по телу — `stopPropagation`.
- **Confirm-диалог** для необратимого: блок/разблок, удаление промокода/кнопки,
  перевыдача подписки, обновление, перезапуски, миграция.
- **Drawer** пользователя: overlay + `slideIn`, ESC закрывает.
- **Живые процессы** (в реале — polling/websocket статуса задачи): прогресс рассылки,
  прогресс миграции, спиннер синхронизации Remnawave.
- **Copy-to-clipboard** для UTM-ссылок и реф-ссылок (`navigator.clipboard`) + toast.
- **Поиск настроек** — дебаунс необязателен на мок-данных, но желателен на API.

## STATE MANAGEMENT
Прототип держит всё в одном компоненте. В реале разнести:
- **Серверный стейт (react-query):** users, payments, promos, groups, campaigns,
  broadcasts, tickets, plans/periods/trafPacks, servers, params (настройки),
  providers, holidays, menu (кнопки бота), topics. Мутации → invalidate.
- **UI-стейт:** section (роут), theme, lang, поисковые строки/фильтры (q, uFilter,
  payFilter, setQ), выбранные сущности (selUser, selTicket, selNode), активные табы
  (uTab, tariffTab, payTab), черновики форм (draft промокода/плана/кампании),
  `dirty`-мапа изменённых настроек, состояния процессов (spin, migProg, прогресс
  рассылок), открытая модалка/confirm/toast.
- **Локальные формы:** значения полей до сохранения; на «Сохранить» — PATCH и optimistic
  update; «Отменить» — сброс `dirty`.

## DATA MODELS (ключевые поля — из прототипа)
- **User** `{id, tg, un(@username), name, st: active|trial|expired|blocked, sub, till,
  bal, traf, dev, reg, last, srv, pg(промогруппа), refInv, refEarn, hwid}`
- **Payment** `{id(TX-...), user, amt, prov, st: ok|pend|fail|refund, date, desc}`
- **Promo** `{code, type: balance|days|trial|group, val, used, lim(0=∞), till, on}`
- **PromoGroup** `{n, d, disc, users, auto}`  •  **Plan** `{id,name,days,price,traf,dev,on,sales}`
- **Period** `{d,p,on}`  •  **TrafPack** `{n,p,on}`
- **Campaign** `{id,name,par,created,regs,trials,paid,rev,group,on}` (+ вычисл. CPA/ROI/CR)
- **Broadcast** `{id,title,aud,date,total,sent,fail,st: run|done,prog}`
- **Ticket** `{id,un,subj,st: open|wait|closed,time, msgs:[{who:u|a,t,tm}]}`
- **Server** `{id,name,loc,load,users,traf,ping,up,on,st: online|maint}`
- **Provider** `{id,name,on,fee(%),bal}` (18 шт.)
- **MenuNode** `{id,parent,label,kind: screen|action|link|miniapp|back,text,emoji(ID),color(HEX)}`
- **Holiday** `{id,d(ДД.ММ),name,on,type: disc|days|bal,val,past,res}`
- **Topic** `{id,name,desc,tid,sched,on}`
- **Param** `{cat,key,d(описание),type: bool|num|text|secret,v}` — категории:
  main|subs|pay|notif|ref|sec|backup|ui.
  **Словарь названий RU (key → человеческое имя)** — использовать в UI:
  `BOT_TOKEN`→Токен бота, `ADMIN_IDS`→Администраторы, `SUPPORT_CHAT_ID`→Группа
  поддержки, `MAINTENANCE_MODE`→Режим техработ, `LANGUAGE_SELECTION_ENABLED`→Выбор
  языка при старте, `TRIAL_ENABLED`→Пробный период, `TRIAL_DURATION_DAYS`→Длительность
  триала, `TRIAL_TRAFFIC_GB`→Трафик на триале, `DEFAULT_DEVICE_LIMIT`→Устройств по
  умолчанию, `AUTO_RENEWAL_ENABLED`→Автопродление с баланса,
  `TRIAL_ADD_REMAINING_DAYS_TO_PAID`→Перенос остатка триала, `MIN_DEPOSIT_AMOUNT`→
  Минимальное пополнение, `YOOKASSA_ENABLED`→Платежи YooKassa, `CRYPTOBOT_ENABLED`→
  Платежи CryptoBot, `TELEGRAM_STARS_ENABLED`→Оплата Telegram Stars, `RECEIPTS_ENABLED`
  →Чеки самозанятого (NaloGO), `EXPIRY_WARNING_DAYS`→Предупреждение об истечении,
  `TRAFFIC_ALERT_PERCENT`→Порог уведомления о трафике, `PAYMENT_NOTIFICATIONS_TOPIC_ID`
  →Топик уведомлений о платежах, `ADMIN_ALERTS_ENABLED`→Алерты администраторам,
  `REFERRAL_ENABLED`→Реферальная программа, `REFERRAL_BONUS_RUB`→Бонус за приглашённого,
  `REFERRAL_PERCENT`→Процент с платежей рефералов, `BLACKLIST_CHECK_ENABLED`→Проверка по
  чёрному списку, `BLACKLIST_UPDATE_INTERVAL_HOURS`→Обновление чёрного списка,
  `TRAFFIC_MONITORING_ENABLED`→Мониторинг аномального трафика,
  `TRAFFIC_THRESHOLD_GB_PER_DAY`→Порог аномалии трафика, `CHANNEL_SUB_REQUIRED`→
  Обязательная подписка на канал, `BACKUP_ENABLED`→Автоматический бэкап,
  `BACKUP_INTERVAL_HOURS`→Интервал бэкапа, `BACKUP_KEEP_LAST`→Хранить копий,
  `MAIN_MENU_MODE`→Режим главного меню, `CONNECT_BUTTON_HAPP_REDIRECT_ENABLED`→
  HTTPS-редирект для Happ, `HIDE_SUBSCRIPTION_LINK`→Скрывать ссылку подписки.
