# 0001 — Слоистая архитектура (4 кольца, протоколы)

**Статус:** Accepted

## Контекст

Спектр решений: строгая Clean Architecture (чисто, но переинженерено: Interactor-на-операцию)
против прагматичной слоистости (широко, но без дисциплины разрастается: сотни сервисов,
god-object PaymentService). Нам нужен фундамент под долгую разработку вдвоём + AI.

## Решение

Четыре кольца с зависимостями внутрь: `core → application → infrastructure/web`.
`application/common` держит **протоколы** (интерфейсы); `application/services` — бизнес-логику
через эти протоколы. Инфраструктура реализует протоколы. **Без** церемонии «один Interactor = один
файл» — композируемые методы сервисов; командный объект только для платёжно-покупочного пайплайна.

## Последствия

- (+) Бизнес-логика тестируется с фейками, не привязана к aiogram/FastAPI.
- (+) Легко подключить бота и миниаппу поверх (швы готовы).
- (+) Понятная карта кода для нового человека/AI.
- (−) Чуть больше бойлерплейта (протоколы + DI), чем в «прямых» хендлерах.
- Enforcement: `application/services` не импортирует `infrastructure`. Нарушение = нужен новый протокол.
