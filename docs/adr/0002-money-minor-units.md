# 0002 — Деньги — целые minor-units

**Статус:** Accepted

## Контекст

Float для денег даёт дрейф округления; крипта/Stars добавляют FX-неточность. Проверенная
практика — хранить всё в целых копейках.

## Решение

Внутри системы деньги — **целые minor-units** (копейки/центы), тип `core/money.Money`
(`amount_minor: int` + `currency`). Конвертация в `Decimal` (ROUND_HALF_UP) — **только** на границе
платёжного шлюза. Для крипты/Stars — допускается `skip_amount_check`-толеранс на сверку сумм.

## Последствия

- (+) Нет дрейфа float; арифметика скидок/рефералки точная.
- (+) Простое хранение (`BIGINT`), простые сравнения.
- (−) Нужно помнить про конверсию minor↔major на UI и на границе шлюза.
- Все денежные колонки БД — `*_minor BIGINT`.
