# ИИ-поддержка

Нейросеть отвечает клиентам в [тикетах](/bot/tickets) вместо оператора: представляется сотрудником поддержки, сверяется с подпиской клиента, решает типовые проблемы реальными действиями (сброс устройств, проверка оплаты) — а всё, что касается денег, споров или выходит за пределы её знаний, передаёт живому человеку. Работает и в боте, и в чате поддержки мини-аппы — движок один.

Ключ к нейросети — ваш собственный: каждый владелец вставляет свой API-ключ Claude, ничего не захардкожено и не проходит через третьи руки.

## Как отвечает

На новое сообщение в тикете ИИ получает историю переписки, базу знаний и живой контекст клиента: статус и срок подписки, пробный период, число подключённых устройств. Отвечает формально, на «вы», без markdown-разметки, короткими конкретными шагами. Ответ сохраняется в тикет с пометкой 🤖 и переводит его в статус «ждём юзера».

По ходу ответа ИИ может выполнять реальные действия — инструменты дергают панель и базу напрямую:

| Инструмент | Что делает |
|---|---|
| `reset_devices` | Сбрасывает все привязанные устройства клиента — при жалобе «превышено число устройств» |
| `list_devices` | Показывает список подключённых устройств (только чтение) |
| `check_payment` | Статус последней оплаты: прошла / в ожидании / нет |
| `check_subscription` | Создана ли подписка, есть ли она в панели, ссылка-подписка |
| `report_to_owner` | Баг-репорт владельцу в топик «alerts» (например: оплата прошла, а подписка не создалась) |

Настройка «Скрывать ссылку подписки» уважается: если `HIDE_SUBSCRIPTION_LINK` включён, ИИ не получает сырой URL и не разгласит его.

## Когда эскалирует

ИИ передаёт тикет человеку, если: клиент явно просит живого оператора; вопрос про возврат денег, спор, жалобу или грубость; проблема не решается или точного решения нет в базе знаний; обращение не по теме (реклама, спам). Клиент при этом видит «Приняли обращение — подключаем оператора», а владельцу в топик «alerts» уходит алерт «Тикет #N эскалирован ИИ — нужен оператор».

Встроенные предохранители:

- **Человек в тикете — ИИ молчит.** Как только оператор ответил из кабинета, ИИ навсегда выходит из этого тикета.
- **Лимит раундов.** После 8 ответов ИИ подряд без вмешательства человека тикет принудительно передаётся оператору — это ограничивает расход вашего API-ключа.
- **Сбой — не тишина.** Ошибка API или пустой ответ превращаются в эскалацию: клиент получает подтверждение, тикет остаётся в очереди. Флоу тикетов от ИИ не зависит.
- **Запреты в промпте.** ИИ не обещает возвраты, компенсации и скидки, не выдумывает фактов и не обсуждает внутреннее устройство сервиса.

## Настройка в кабинете

Раздел **«ИИ-поддержка»** кабинета. Порядок:

1. Возьмите API-ключ (`sk-ant-…`) на [console.anthropic.com](https://console.anthropic.com) и вставьте в поле ключа. Хранится в зашифрованном виде — обратно из кабинета и API не читается, только отметка «заполнено».
2. Модель менять не обязательно — по умолчанию `claude-haiku-4-5-20251001` (дёшево и быстро для поддержки).
3. Опишите свой сервис в «Базе знаний»: тарифы, какие приложения ставить, типовые решения. Пусто — используется встроенный набор (подключение, тарифы, устройства, оплата, «не подключается»).
4. Нажмите **«Тест»** — задайте пробный вопрос и посмотрите ответ до включения на реальных клиентах. Тест безопасен: работает на синтетическом клиенте без подписки, устройства не сбрасывает, алерты не шлёт.
5. Включите тумблер **«Включить ИИ-поддержку»**.

| Параметр | По умолчанию | Что это |
|---|---|---|
| `AI_SUPPORT_ENABLED` | выкл | Тумблер. ИИ активен только при включённом тумблере **и** заполненном ключе |
| `AI_SUPPORT_API_KEY` | — | Ваш ключ Claude с console.anthropic.com, шифруется |
| `AI_SUPPORT_MODEL` | `claude-haiku-4-5-20251001` | Модель Claude |
| `AI_SUPPORT_KNOWLEDGE_BASE` | встроенный набор | Факты о сервисе — этим «обучаете» ИИ (до 20 000 символов) |
| `AI_SUPPORT_EXTRA_PROMPT` | — | Доп. инструкции: тон, стиль, особые правила (до 4 000 символов) |

::: tip Стоимость под контролем
Стабильная часть запроса (промпт + база знаний + инструменты) кэшируется на стороне API — повторные ответы обходятся примерно в 10 раз дешевле. Плюс лимит в 8 раундов на тикет не даёт диалогу крутиться бесконечно.
:::

::: warning Ротация ключа
Если ключ утёк или вы передаёте проект другому человеку — отзовите старый ключ на console.anthropic.com и впишите новый в разделе «ИИ-поддержка». Ключ в базе зашифрован, но отзыв на стороне консоли — единственный способ гарантированно закрыть доступ.
:::
