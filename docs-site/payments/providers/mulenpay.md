# MulenPay

MulenPay (`mulenpay.ru`) — оплата картой и по СБП. База создаёт платёж запросом `POST /api/v2/payments`, юзер уходит на платёжную страницу из ответа (`paymentUrl`). Валюта — только RUB, платёжная страница открывается на русском.

Обе стороны обмена подписаны SHA1: исходящий запрос — одной формулой, входящий вебхук — другой, обе на базе `secret_key`.

## Что понадобится

Кабинет MulenPay, из него три значения:

- **API-ключ** — Bearer-авторизация запросов;
- **ID магазина**;
- **Секретный ключ** — участвует в подписях запроса и вебхука.

## Настройка в кабинете

Кабинет → Платежи → карточка MulenPay:

| Поле | Что это | Обязательное |
|---|---|---|
| `api_key` | API-ключ (Bearer) для создания платежей | да |
| `shop_id` | ID магазина | да |
| `secret_key` | Секретный ключ — SHA1-подписи запроса и вебхука | да |

Ключи шифруются при сохранении. Включи провайдера тумблером — выключенный отвечает на вебхуки 404. Кнопка «Тест» для MulenPay API-пробу не выполняет — проверяй первым платежом. Общие поля карточки — в [настройках платежей](/payments/settings).

## Вебхук

URL уведомлений в кабинете MulenPay:

```
https://ваш-домен/api/v1/payments/mulenpay
```

Уведомление — JSON с полем `sign` внутри тела. Проверка:

```
sign = sha1(конкатенация значений тела без sign, в порядке следования полей + secret_key)
```

Неверная подпись → 403. IP-фильтра нет.

Статусы из уведомления: `paid`/`success`/`succeeded`/`3` — оплачен, `canceled`/`fail`/`failed`/`expired` — отменён, всё остальное — остаётся в ожидании.

## Особенности

- Возвратов по API и рекуррентов нет — возврат фиксируется как ручной, см. [Возвраты](/payments/refunds).
- Позиции чека при создании платежа не передаются (`items` уходит пустым) — фискализацию, если она нужна, настраивай на стороне MulenPay.
- Подпись запроса создания: `sha1(currency + amount + shop_id + secret_key)` — если платежи не создаются с ошибкой подписи, первым проверяй `secret_key`.
- Опроса статуса по API нет: потерянный вебхук оставляет платёж в ожидании.

::: tip Платёж не пришёл?
Реконсилятор проверяет зависшие платежи каждые 5 минут, но MulenPay не даёт опроса статуса — оплату закрывает только вебхук. Проверь URL уведомлений — разбор доставки в разделе [Вебхуки](/payments/webhooks).
:::
