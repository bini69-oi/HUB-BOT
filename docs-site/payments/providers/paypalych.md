# PayPalych

PayPalych (`pal24.pro`) — выставление счетов (bills) с оплатой картой и по СБП. База создаёт счёт запросом `POST /api/v1/bill/create`, юзер уходит на платёжную страницу из ответа. Валюта — только RUB.

Уведомления приходят в Robokassa-подобном формате с MD5-подписью. Плюс этого шлюза — есть API опроса статуса счёта, так что потерянный вебхук не оставляет платёж висеть.

## Что понадобится

Кабинет pal24.pro. Из него:

- **API-токен** — Bearer-авторизация запросов;
- **ID магазина**;
- опционально — отдельный **токен подписи** уведомлений, если он у тебя отличается от API-токена.

## Настройка в кабинете

Кабинет → Платежи → карточка PayPalych:

| Поле | Что это | Обязательное |
|---|---|---|
| `api_token` | API-токен (Bearer) для создания счетов и опроса статуса | да |
| `shop_id` | ID магазина | да |
| `signature_token` | Токен подписи вебхука; если пусто — используется `api_token` | нет |

Токены шифруются при сохранении. Включи провайдера тумблером — выключенный отвечает на вебхуки 404. Кнопка «Тест» для PayPalych API-пробу не выполняет — проверяй первым платежом. Общие поля карточки — в [настройках платежей](/payments/settings).

## Вебхук

URL уведомлений в кабинете PayPalych:

```
https://ваш-домен/api/v1/payments/paypalych
```

Приходит POST form-encoded (некоторые конфигурации шлют JSON — база понимает оба варианта) с полями `OutSum`, `InvId`, `Status`, `SignatureValue`. Подпись:

```
SignatureValue = MD5(OutSum:InvId:token).upper()
```

где `token` — `signature_token` (или `api_token`, если отдельный не задан), а `InvId` — id счёта PayPalych. Неверная подпись → 403. IP-фильтра нет.

Статусы из уведомления: `SUCCESS` и `OVERPAID` — оплачен, `FAIL`/`CANCELLED` — отменён, `NEW`/`PROCESS`/`UNDERPAID` — остаётся в ожидании.

## Особенности

- Переплата (`OVERPAID`) засчитывается как оплата, недоплата (`UNDERPAID`) — нет, счёт продолжает ждать.
- Есть опрос статуса `GET /api/v1/bill/status` — реконсилятор сам добивает платежи, чей вебхук потерялся.
- Возвратов по API и рекуррентов нет — возврат фиксируется как ручной, см. [Возвраты](/payments/refunds).
- Сумма транзакции фиксируется из `OutSum` уведомления; описание платежа обрезается до 128 символов при создании счёта.

::: tip Платёж не пришёл?
PayPalych поддерживает опрос статуса — реконсилятор проверяет зависшие платежи каждые 5 минут и добьёт оплату даже без вебхука. Как это устроено — в разделе [Вебхуки](/payments/webhooks).
:::
