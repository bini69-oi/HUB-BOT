# Установка одной командой

Никаких `.env` руками. Скрипт задаёт пять вопросов, всё остальное делает сам: ставит Docker, генерирует секреты, поднимает весь стек и отдаёт ссылку на кабинет. Дальнейшая настройка — тарифы, платёжки, меню бота — целиком в веб-UI.

На чистом VPS (Ubuntu/Debian):

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/bini69-oi/HUB-BOT/main/scripts/install.sh)
```

То же из клона репозитория:

```bash
git clone https://github.com/bini69-oi/HUB-BOT.git && cd HUB-BOT
./scripts/install.sh
```

## Требования к серверу

| Что | Сколько |
|---|---|
| CPU / RAM | 1 vCPU / 1–2 GB (на 1 GB скрипт сам поднимет swap) |
| ОС | Ubuntu/Debian (git ставится через `apt-get`, Docker — через get.docker.com) |
| Права | root — для установки Docker и создания swap |
| Домен | опционально: с доменом кабинет получает HTTPS автоматически, без — работает по `http://IP` |

## Что спросит скрипт

| Вопрос | Обязательно | Если нажать Enter |
|---|---|---|
| Токен бота из @BotFather | да | — |
| Домен для кабинета | нет | кабинет на `http://IP`, без HTTPS |
| E-mail для Let's Encrypt | только если указан домен | — |
| URL панели Remnawave | нет | поднимется встроенная [мок-панель](/panel/mock) — стек можно пощупать целиком |
| API-токен панели | только если указан URL панели | — |
| Ваш Telegram ID (узнать — @userinfobot) | нет | без личных уведомлений владельцу (панель упала, бэкап, споры) |

## Что скрипт делает сам

1. При запуске через `curl` — ставит git (если нет) и клонирует репозиторий в папку `HUB-BOT`.
2. Ставит Docker через get.docker.com, если его нет; проверяет наличие docker compose v2.
3. Проверяет память: если RAM меньше ~1.8 GB и swap отсутствует — создаёт swap-файл 2 GB (`/swapfile`, с записью в `/etc/fstab`), чтобы сборка образов не упала по OOM.
4. Генерирует секреты (ключ шифрования, JWT, пароль БД, пароль админа, вебхук-секрет) и пишет `.env` с правами 600.
5. Если панель не указана — включает профиль мок-панели (`COMPOSE_PROFILES=mock` в `.env`).
6. Собирает и запускает стек: `postgres · redis · web · bot · worker · scheduler · caddy` (плюс `mockpanel`, если панели нет). Первая сборка — несколько минут.
7. Ждёт `/health` до 3 минут — миграции БД web-контейнер применяет сам при старте.

С доменом HTTPS поднимается автоматически: Caddy получает сертификат Let's Encrypt на указанный e-mail.

## Что выдаст в конце

- **Кабинет** — `https://ваш-домен/admin/` (или `http://IP/admin/`), логин `admin` и сгенерированный пароль.
- **Мини-аппа** — `…/app/`.
- **Пароль шифрования бэкапов** — им зашифрованы бэкапы БД, пока вы не зададите свой в кабинете → [Обслуживание](/cabinet/maintenance).

::: warning Пароль бэкапов — сохранить отдельно
Запишите пароль шифрования бэкапов ОТДЕЛЬНО от сервера. Без него бэкап БД не расшифровать — сервер умер вместе с паролем, значит умерли и бэкапы. Подробнее — [Бэкапы](/features/backups).
:::

::: tip Повторный запуск
Если `.env` уже существует, скрипт использует его и не задаёт вопросов — установку можно безопасно перезапускать. Для чистой установки удалите `.env`.
:::

Если что-то пошло не так, скрипт печатает хвост лога и путь к полному логу; логи стека — `docker compose -f docker/compose.prod.yml logs web`.
