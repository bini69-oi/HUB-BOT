# Промо и акции

Раздел «Акции и промокоды» — промокоды, промогруппы со скидками и сводка рефералки. Календарь акций живёт на соседнем экране «Автоуведомления». Как промокоды выглядят со стороны юзера — [/bot/promo](/bot/promo).

## Промокоды

Код задаётся вручную или генерится автоматически (8 символов, буквы + цифры). Четыре типа награды:

| Тип | Что получает юзер |
|---|---|
| Баланс | Пополнение кошелька на номинал (в форме кабинета номинал вводится в рублях) |
| Дни | +N дней к **активной** подписке — без действующей подписки код не применится |
| Триал | Бесплатная подписка на N дней: действующая продлевается, без подписки — выдаётся новая (тем же panel-first путём, что и триал) |
| Группа | Юзер добавляется в промогруппу — получает её скидки |

У кода настраиваются лимит активаций (0 — безлимит) и срок действия; код можно выключить не удаляя. Один юзер активирует код один раз, подсчёт активаций атомарный — «перебрать лимит» параллельными активациями нельзя. Дни и выдача подписки применяются панель-first: сначала Remnawave, потом запись в базе.

## Пачки gift-кодов (CSV)

Кнопка массовой генерации создаёт до 1000 подарочных кодов за раз: задаёте количество и номинал в днях, каждый код получает префикс `GIFT` и **жёстко одноразовый** (одна активация). Многоразовые кампании создаются обычной формой с явным лимитом.

Кабинет сразу скачивает CSV с двумя колонками:

```csv
code,gift_link
GIFT-7K2M9QRX,https://t.me/your_bot?start=gift_GIFT-7K2M9QRX
```

`gift_link` — deep-link прямо в бота: юзер нажимает ссылку, код активируется на старте. Ссылки собираются из настройки `BOT_USERNAME` — если она пустая, колонка останется без ссылок.

::: tip Куда девать пачку
Такой CSV удобно раздавать через рекламные посевы, вкладывать в рассылки или отдавать блогеру: каждый код — трекаемый одноразовый подарок.
:::

## Промогруппы

Промогруппа — набор скидок, привязанный к юзерам:

| Поле | Смысл |
|---|---|
| Приоритет | У юзера в нескольких группах действует группа с наибольшим приоритетом |
| Скидка на серверы, % | Базовый процент группы |
| Скидки за период | Отдельный % на конкретные длительности, например `{"90": 10, "180": 20}` |
| Порог автоназначения | Сумма пожизненных трат, при достижении которой группа выдаётся автоматически |
| Группа по умолчанию | Помечает дефолтную группу |

Скидка группы участвует в общем расчёте цены вместе с персональной и одноразовой (кап — 100%, подробнее в [/cabinet/tariffs](/cabinet/tariffs)). В группу юзер попадает и промокодом типа «Группа» — удобно продавать «клубные» скидки.

## Календарь акций

На экране «Автоуведомления» есть «Календарь акций · праздники РФ»: при первом открытии автоматически создаётся набор праздников (Новый год, Рождество, 23 февраля и далее по календарю). Для каждой даты настраивается:

- включена ли рассылка;
- тип награды: скидка %, дни или баланс;
- значение и время отправки (МСК);
- после отправки видны результаты.

Подробнее про автоуведомления и winback — [/cabinet/reminders](/cabinet/reminders).

## Рефералка

Внизу раздела — сводка реферальной программы: включена ли, бонус за приглашённого, процент с платежей, всего приведено и выплачено. Сами параметры меняются в [/cabinet/settings](/cabinet/settings), механика описана в [/bot/referral](/bot/referral).
