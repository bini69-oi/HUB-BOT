# Рекламные кампании

Учёт платного трафика через диплинки: каждой кампании — своя метка `?start=`, кабинет считает регистрации, триалы, платящих и выручку по метке и сводит это в CR, CPA и ROI. Для арбитража есть S2S-постбеки в трекер.

## Создание кампании

1. Задайте название и метку `start_param` — латиница, цифры, `_` и `-`, от 2 до 64 символов, уникальна среди кампаний.
2. При необходимости привяжите промогруппу и укажите расход на кампанию (для CPA/ROI).
3. Кабинет сразу покажет готовую ссылку вида `https://t.me/<бот>?start=<метка>` — её и размещаете.

::: warning Зарезервированные префиксы
Не начинайте метку с `ref_`, `partner_` или `gift_` — эти префиксы диплинков заняты реферальной программой, партнёрскими ссылками и подарочными промокодами (см. [Рефералка](/bot/referral) и [Промокоды](/bot/promo)). Такой параметр уйдёт в другую ветку обработки и в кампанию не запишется.
:::

## Как работает атрибуция

Пользователь открывает бота по ссылке `t.me/<бот>?start=<метка>` — обработчик `/start` находит **активную** кампанию с такой меткой и записывает её пользователю. Правила:

- **First touch:** метка записывается только пользователям, у которых кампании ещё нет, — и новым, и старым без метки. Повторные переходы по другим ссылкам атрибуцию не перезаписывают.
- Кампания на паузе (`is_active = false`) новых пользователей не атрибуцирует; уже собранная статистика остаётся.
- Если к кампании привязана промогруппа, пользователь автоматически в неё попадает — так трафику с конкретного источника выдаются особые цены (см. [Промогруппы и скидки](/cabinet/promos)).

## Статистика по кампаниям

Метрики считаются по пользователям, атрибуцированным к кампании:

| Метрика | Как считается |
|---|---|
| Регистрации | Пользователи с меткой кампании |
| Триалы | Выданные триал-подписки среди них |
| Платящие | Уникальные пользователи с завершёнными платежами (пополнения + оплаты подписки) |
| Выручка | Сумма этих платежей |
| CR % | Платящие / регистрации × 100 |
| CPA | Расход кампании / платящие |
| ROI % | (Выручка − расход) / расход × 100 |
| Средний чек | Выручка / платящие |

Расход (`cost`) вводится вручную и правится в любой момент — CPA и ROI пересчитываются от него. Кампанию можно поставить на паузу, переименовать или удалить.

## S2S-постбеки для арбитража

Бот умеет дёргать трекер сервер-к-серверу на трёх событиях: регистрация, выдача триала, покупка. Постбек — это GET-запрос на URL-шаблон из настроек, макросы подставляются перед отправкой.

Настраивается в [Настройках](/cabinet/settings):

| Параметр | По умолчанию | Что это |
|---|---|---|
| `POSTBACK_ENABLED` | выкл | Главный выключатель |
| `POSTBACK_URL_REGISTRATION` | пусто | URL-шаблон на регистрацию |
| `POSTBACK_URL_TRIAL` | пусто | URL-шаблон на триал |
| `POSTBACK_URL_PURCHASE` | пусто | URL-шаблон на покупку |

Макросы в шаблоне:

| Макрос | Значение |
|---|---|
| `{event}` | `registration` / `trial` / `purchase` |
| `{user_id}` | Внутренний ID пользователя |
| `{tg_id}` | Telegram ID (заполняется только на регистрации) |
| `{amount}` | Сумма покупки в рублях с копейками (`299.00`); `0` для регистрации и триала |
| `{subid}` | Метка кампании пользователя (`start_param`); пусто, если пользователь пришёл не с кампании |

Пример шаблона на покупку:

```text
https://tracker.example.com/postback?cid={subid}&payout={amount}&event={event}
```

Постбеки отправляются best-effort с таймаутом 10 секунд: лежащий трекер никак не влияет на работу бота, неудача просто пишется в лог.
