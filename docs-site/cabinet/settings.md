# Настройки

Экран «Все настройки» — hot-reload параметры бота и магазина. Здесь живёт всё, что раньше требовало правки конфига и рестарта: триал, рефералка, платежи, поддержка, баннеры.

![Все настройки](/assets/admin-settings.png)

## Как это устроено

Каждый параметр объявлен в реестре в коде: ключ, секция, тип (переключатель / число / строка / секрет), значение по умолчанию и описание. База хранит **только переопределения** — то, что вы меняли руками. Поэтому:

- обновление базы не трогает ваши настройки, а новые параметры появляются в кабинете сами, без миграций;
- кнопка сброса возвращает заводское значение (просто удаляет переопределение), изменённые параметры помечены бейджем «ИЗМЕНЕНО».

Изменения применяются **без рестарта**: веб-процесс подхватывает их мгновенно, бот и воркер — в пределах 15 секунд (кэш конфигурации самообновляется). Кабинет так и подтверждает сохранение: «Применено без перезапуска».

Поиск наверху экрана ищет по ключу, названию и описанию. Тот же поиск встроен в сайдбар кабинета — совпавшая настройка открывается сразу в нужной секции.

::: warning Секреты шифруются
Параметры-секреты (пароли SMTP, API-ключи, токены) рендерятся как поля пароля, в списке маскируются (`••••••••`) и хранятся в базе в зашифрованном виде — Fernet-ключом `APP__CRYPT_KEY`. Если сменить `APP__CRYPT_KEY`, старые секреты станут нечитаемыми и параметры откатятся к дефолту — их придётся ввести заново.
:::

Соглашения: денежные значения — в копейках (minor units), длительности — в днях/часах, время — `HH:MM` по МСК.

## Секции и ключевые параметры

Всего параметров около сотни, ниже — самые важные по секциям.

### Основные

| Параметр | Дефолт | Зачем |
|---|---|---|
| `BOT_USERNAME` | — | Username бота без @; из него собираются все deep-links (рефералка, gift-коды, кампании) |
| `ADMIN_IDS` | — | Telegram ID через запятую — кому доступна админка в боте |
| `MAINTENANCE_MODE` | выкл | Бот отвечает заглушкой всем, кроме админов |
| `AUTO_PURCHASE_AFTER_TOPUP` | вкл | Не хватило на балансе → пополнил → покупка завершается сама |
| `MAIN_MENU_MODE` | `inline` | Кнопки под сообщением или reply-клавиатура |

### Подписки и триал

| Параметр | Дефолт | Зачем |
|---|---|---|
| `SALES_MODE` | `plans` | Каталог планов или конструктор — см. [/cabinet/tariffs](/cabinet/tariffs) |
| `TRIAL_ENABLED` | вкл | Пробный период |
| `TRIAL_DURATION_DAYS` / `TRIAL_TRAFFIC_GB` / `TRIAL_DEVICE_LIMIT` | 3 / 10 / 1 | Параметры триала (0 ГБ — безлимит) |
| `TRIAL_PRICE` | 0 | Платный триал: цена в копейках, 0 — бесплатно |
| `TRIAL_CARRYOVER_DAYS` | вкл | Неиспользованные дни триала добавляются к первой оплате |
| `AUTO_RENEWAL_ENABLED` | вкл | Автопродление с баланса до истечения (за `AUTO_RENEWAL_DAYS_BEFORE` дней) |

### Платежи

| Параметр | Дефолт | Зачем |
|---|---|---|
| `MIN_DEPOSIT_AMOUNT` | 5000 коп. (50 ₽) | Минимальное пополнение баланса |
| `BALANCE_ENABLED` | вкл | Кошелёк-баланс: пополнение и оплата с него |
| `STARS_RATE_RUB` | 130 коп. за ★ | Курс пересчёта цен в Telegram Stars — см. [/payments/providers/telegram-stars](/payments/providers/telegram-stars) |
| `TAX_RATE_PERCENT` | 6 | Ставка налога для расчёта чистой прибыли в разделе «Платежи» |
| `REFUND_ENABLED` | выкл | Возвраты из кабинета — см. [/payments/refunds](/payments/refunds) |
| `NALOGO_ENABLED` | выкл | Чеки «Мой налог» для самозанятых (ИНН + device-токен) |

Подключение самих касс — в [/payments/settings](/payments/settings).

### Реферальная программа

| Параметр | Дефолт | Зачем |
|---|---|---|
| `REFERRAL_ENABLED` | вкл | Программа в целом — механика в [/bot/referral](/bot/referral) |
| `REFERRAL_BONUS_RUB` | 5000 коп. (50 ₽) | Бонус пригласившему после первой оплаты друга |
| `REFERRAL_BONUS_DAYS` | 7 | +N дней подписки обоим |
| `REFERRAL_PERCENT` | 10 | Доля с каждого пополнения реферала |
| `REFERRAL_WITHDRAWAL_ENABLED` | выкл | Вывод реф-заработка: заявка админу, минимум `REFERRAL_WITHDRAWAL_MIN` (500 ₽) |

### Безопасность

| Параметр | Дефолт | Зачем |
|---|---|---|
| `AUTO_MAINTENANCE_ENABLED` | вкл | 3 неудачных пинга панели подряд → техрежим сам включается, восстановление снимает его |
| `REMNAWAVE_RESYNC_ENABLED` | выкл | Ночная сверка подписок с панелью, чинит дрейф от ручных правок — см. [/panel/watchdog](/panel/watchdog) |
| `DEVICE_GUARD_ENABLED` | выкл | Детект шеринга по уникальным IP — см. [/features/device-guard](/features/device-guard) |
| `CHANNEL_SUB_REQUIRED` | выкл | Обязательная подписка на канал — см. [/bot/channel-gate](/bot/channel-gate) |
| `RATE_LIMIT_ENABLED` | выкл | Антифлуд: игнор слишком частых действий одного юзера |

### Уведомления и бэкапы

| Параметр | Дефолт | Зачем |
|---|---|---|
| `REPORT_GROUP_ID` | — | Форум-группа, куда бот пишет отчёты по топикам |
| `REPORT_DM_ADMINS` | выкл | Дублировать отчёты в личку админам |
| `BACKUP_ENABLED` / `BACKUP_TIME` / `BACKUP_KEEP_LAST` | вкл / 04:00 / 7 | Автобэкап базы — см. [/features/backups](/features/backups) |
| `BACKUP_ENCRYPTION_PASSWORD` | — | Пароль шифрования бэкапа (секрет) |

### Интерфейс бота

| Параметр | Дефолт | Зачем |

| `SUPPORT_MODE` | `tickets` | Канал поддержки: тикеты в боте / redirect на аккаунт / отдельный бот / чат в миниаппе |

| `MTPROTO_PROXY_ENABLED` | выкл | Кнопка MTProto-прокси в боте и миниаппе |
