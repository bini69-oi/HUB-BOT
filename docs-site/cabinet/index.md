# Админ-кабинет — обзор

Веб-кабинет — главный пульт управления магазином. SPA раздаётся по адресу `https://ваш-домен/admin`, всё API живёт под `/api/admin/*`. Тарифы, промо, рассылки, настройки бота и миниаппы правятся отсюда без захода на сервер: почти все изменения применяются без рестарта.

![Экраны админ-кабинета](/assets/admin-collage.png)

## Вход

Суперадмин создаётся автоматически при старте приложения из переменных окружения:

```bash
ADMIN__USERNAME=owner        # логин (без @)
ADMIN__PASSWORD=strongpass   # пароль
```

Если оба значения заданы, при каждом старте база проверяет: аккаунта нет — создаётся с ролью OWNER; пароль в env изменился — хэш перезаписывается. Так пароль меняется простым редактированием `.env` + рестартом.

Логин по паре логин/пароль (`POST /api/admin/auth/login`) выдаёт JWT-токен, подписанный `APP__JWT_SECRET`. Срок жизни сессии — `ADMIN__SESSION_TTL_HOURS`, по умолчанию 12 часов. Неверный логин и неверный пароль возвращают одну и ту же ошибку — перебор логинов не работает.

В кабинет пускают только аккаунты со staff-ролью (ADMIN и выше) в статусе ACTIVE. Забаненному или разжалованному админу токен перестаёт действовать сразу — роль и статус проверяются на каждом запросе.

::: info Демо-режим
`ADMIN__DEMO_ENABLED=true` включает кнопку «Демо-режим (только просмотр)» на экране логина: сессия с ролью PREVIEW видит все разделы, но любой изменяющий запрос отбивается с 403. Так устроено публичное демо кабинета. По умолчанию выключено.
:::

## Дашборд

![Дашборд](/assets/admin-dashboard.png)

Первый экран после входа — сводка по бизнесу:

| Блок | Что показывает |
|---|---|
| Выручка сегодня | Сумма за день + сравнение со вчера. Считаются только внешние деньги: пополнения и оплаты через шлюзы. Покупка с внутреннего баланса не учитывается повторно — эти рубли уже засчитаны при пополнении. Тестовые платежи исключены |
| Активные подписки | Подписки в статусах ACTIVE / TRIAL / LIMITED, рядом — всего пользователей |
| Новых за 24ч | Регистрации и новые триалы за сутки, плюс конверсия триала в оплату |
| Онлайн | Суммарный онлайн по всем нодам панели |
| Выручка · 14 дней | График по дням |
| Источники · 30 дн | Откуда пришли новые юзеры: кампании / рефералы / органика |
| Последние события | Лента аудита — последние 20 действий админов (кто, что, над чем) |
| Система | Здоровье панели Remnawave (версия), Redis, размер PostgreSQL, режим техработ, автобэкап |

Тут же кнопки быстрых действий: «Бэкап сейчас», «Создать рассылку», «Синхронизировать Remnawave», «Создать промокод».
